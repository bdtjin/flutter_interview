# 대칭키 암호화, 비대칭키 암호화의 방식에 대해 설명해보세요.

## 대칭키 암호화
- 암호화와 복호화에 동일한 키를 사용하는 방식
- 방식: 송신자와 수신자가 동일한 하나의 비밀키를 공유한다.
- 장점: 속도가 빠르다.
- 단점: 키를 안전하게 전달하기 어렵다. / 사용자가 늘어날수록 관리해야 할 키의 개수가 기하급수적으로 늘어난다.

## 비대칭키 암호화
- 암호화와 복호화에 서로 다른 키를 사용하는 방식
- 방식: 모든 사용자는 공개키와 개인키 한 쌍을 가진다.
  - 공개키: 누구나 알 수 있는 키
  - 개인키: 본인만 알고 있는 키
- 장점: 키를 안전하게 전달하기 쉽다.
- 단점: 속도가 느리다.

## 장단점 비교
| 구분 | 대칭키 암호화 | 비대칭키 암호화 |
| --- | --- | --- |
| 암호화/복호화 키 | 동일한 키 사용 | 서로 다른 키 사용 |
| 속도 | 빠름 | 느림 |
| 키 전달 | 어려움 | 쉬움 |

## 사용 예시
- 대칭키 암호화: 파일 암호화, 데이터베이스 암호화
- 비대칭키 암호화: HTTPS, SSL/TLS, 디지털 서명

### 네이버 로그인 과정
1. 사용자가 네이버 로그인 버튼을 누름

2. 네이버 로그인 화면이 뜸
보안 포인트: 우리 앱은 사용자의 비밀번호를 알 수 없다.

3. 네이버가 인증번호를 줌
앱에게 일회용 인증 번호를 던져 줌

4. 비밀열쇠
우리 앱이 네이버 서버로 가서 임시 번호와 함께 Client Secret을 보여줌 (비밀 열쇠)

5. 네이버에서 인증번호를 확인하고, 사용자 정보를 우리 앱에게 던져 줌

비대칭키: 3~6번 우리 앱과 네이버가 데이터를 주고받을 때, 암호화해서 사용
대칭키: Client Secret을 이용해서 사용자 정보를 암호화해서 우리 앱에게 던져 줄 때 사용
