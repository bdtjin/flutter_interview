# 서버와 클라이언트에 대해 설명해보세요.

## 클라이언트 (서비스를 사용하는 웹 브라우저)
서비스를 요청하는 주체. 사용하는 스마트폰 앱, 웹 브라우저(크롬,사파리) 등이 해당된다.

## 서버 (서비스를 제공하는 컴퓨터 프로그램, 또는 장치)
요청에 따라 서비스를 제공하는 주체. 데이터 저장, 복잡한 계산, 인증 등을 수행하며 24시간 중단 없이 가공된다.

## 통신 과정
두 시스템은 정해진 약속(프로토콜, HTTP, HTTPS)에 따라 대화한다.
1. Request : 클라이언트가 서버에 요청을 한다.
2. Response : 서버가 클라이언트의 요청에 대한 응답을 한다.
3. Procedssing : 서버는 요청을 확인하고, 데이터 베이스에서 정보를 찾거나 로직을 수행한다.

## 상호작용 방식
1. 사용자가(클라이언트) 웹 브라우저를 열기
웹 서핑을 시작한다. 크롬, 사파리, 인터넷 등 웹 서핑을 하면 웹 브랑우저는 사용자가 입력한 URL을 기반으로 해당 웹 페이지를 요청하기 위해 서버에게 요청을 보낸다.

2. 서버가 응답
서버는 클라이언트(웹 브라우저)의 요청을 받고, 웹 페이지의 내용을 찾고 필요한 데이터나 파일 시스템에서 해당 정보를 가져온다. 마지막으로 클라이언트에게 요청받은 웹 페이지의 내용을 응답으로 전송한다.

3. 클라이언트가 서버 응답을 처리한다.
클라이언트(웹 브라우저)는 서버로부터 받은 응답을 처리하고 화면에 표시한다.
