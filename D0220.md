# Flutter에서 BuildContext란 무엇인지 설명해보세요.

## BuildContext
BuildContext란 UI 트리에서 위젯의 위치와 상태를 식별하는 데 사용되는 개념이다.
다양한 Flutter 위젯 및 함수 내에서 사용되며, 위젯트리를 탐색하고 위젯 간에 정보를 공유하는 데 도움을 준다.
(buildcontext는 현재 빌드에 대한 정볼르 저장하는 객체이다. 모든 부모요소의 정보를 갖고 있는 객체를 의미한다.
그렇기 떄문에 하위 위젯에서 부모 위젯의 정보에 접근할 수 있다.)

## BuildContext와 Element Tree와의 관계
BuildContext는 위젯에 대응하는 Element 그 자체라고 말할 수 있다. 위젯은 설정값일 뿐이고, 실제 트리에 고정되어 상태를 관리하는 것은 Element이므로, BuildContext를 통해 해당 위치의 상태를 관리할 수 있다.
 - Flutter에서는 3가지 트리가 있음. Widget, Element, RenderObject
   - Widget Tree : 위젯의 구조를 나타내는 트리
   - Element Tree : 위젯의 상태를 나타내는 트리
   - RenderObject Tree : 위젯의 렌더링을 나타내는 트리


## 주요 역할 기능
### 위젯 트리 검색
- BuldContext는 현재 위젯의 위치를 나타내며, 위젯 트리에서 부모 위젯과 자식 위젯에 대한 참조를 얻는 데 사용된다. BuildContext를 사용하여 위젯 트리에서 위로올라가거나 아래로 이동하여 특정 위젯을 찾을 수 있다.
ex) Theme.of(context), MediaQuery.of(context)

### 리소스 및 네이게이션 제어
ex) Navigator.of(context).push()를 통해 화면 전환을 한다
ex) Scaffold.of(context).showSnackBar()를 통해 SnackBar를 띄운다

### 렌더링 정보 제공
위젯의 화면상 크기나 위치 정보를 알고 싶을 때, context.findRenderObject()와 같은 메서드를 통해 렌더링 계층에 접근할 수 있다.

### Flutter의 InheritedWidget
트리를 통해 데이터를 하위 위젯으로 효율적으로 전달하는 기본 클래스로 dependOnInheritedWidgetOfExactType을 사용하여 하위 위젯에서 상위의 데이터를 조회하고 업데이트를 구독한다.
