정렬 알고리즘이란 무엇이며, 사용 이유에 대해 설명해주세요.

## 정렬 알고리즘
정렬 알고리즘이란 컴퓨터 과학의 가장 기초적이면서  중요한 주제 중 하나입니다.
무작위로 나열된 데이터 집합을 특정 기준(오름차순 또는 내림차순)에 따라 일정한 순서로 재배열하는 일련의 과정을 의미합니다.

데이터 형태에 따라 숫자 크기순, 문자 사전순 등으로 정렬할 수 있으며, 효율적인 정렬은 이후의 데이터 처리 속도를 결정짓는 핵심적인 선행 단계입니다.

## 정렬 알고리즘을 사용하는 이유
단순히 보기 좋게 나열하는 것을 넘어, 정렬은 시스템의 전반적인 성능 최적화를 위해 필수적입니다.

#### 1. 탐색 성능의 극대화
데이터가 정렬되어 있지 않으면 특정 값을 찾기 위해 처음부터 끝까지 확인하는 성형 탐색을 해야 합니다. 하지만 정렬된 데이터에서는 이진 탐색을 사용할 수 있어 탐색 속도가 획기적으로 빨라집니다.
- 미정렬 상태 : $O(n)$
- 정렬 상태 (이진 탐색) : $O(\log n)$

#### 2. 데이터 처리 효율성 향상
- 중복 제거: 정렬된 데이터에서는 인접한 요소끼리 비교하여 중복된 항목을 쉽게 찾아낼 수 있습니다.
- 그룹화: 특정 기준에 따라 데이터를 모아서 분석하거나 통계를 낼 때 정렬이 되어 있으면 처리가 간결해집니다.

---

정렬 알고리즘을 선택할 때는 단순히 속도 뿐만 아니라, **시간 복잡도**와 **공간 복잡도**, 그리고 동일한 값의 상ㄷ대적 순서가 유지되는지 여부인 **안정성**을 종합적으로 고려해야 합니다.

정렬 알고리즘은 데이터를 특정 기준에 맞춰 재배열하는 논리적인 절차입니다. 정렬을 사용하는 가장 큰 이유는 데이터 처리의 효율성 때문입니다. 특히 데이터가 정렬되어 있으면 이진 탐색과 같은 알고리즘을 통해 탐색 속도를 $O(n)$에서 $O(\log n)$으로 크게 단축할 수 있습니다. 또한 중복 데이터 제거와 같은 전처리 과정을 최적화하여 전체 시스템의 성능을 향상시키기 위해 필수적으로 사용됩니다.

* $O(n)$ 데이터 개수와 처리 시간 정비례
* $O(log n)$ 데이터가 늘어나도 처리 시간은 아주 조금씩만 늘어남

## 정렬 알고리즘 종류
수와 종류가 매우 다양하지만, 보통 효율성, 메모리 사용량, 그리고 데이터의 특성 보존 여부라는 4가지 주요 기준에 따라 분류됩니다.

1. 시간 복잡도
알고리즘이 데이터 정렬하는 걸리는 수행 시간을 기준으로 나눕니다. 데이터의 개수(n)가 늘어날 때 시간이 얼마나 급격히 증가하는지를 나타내는 Big-O 표기법ㅂ을 사용합니다.

- 단순한 정렬: 구현은 쉽지만 대량의 데이터에는 부적합합니다. (예: 버블정렬, 선택 정렬, 삽입 정렬)
- 효율적인 정렬: 복잡하지만 대량의 데이터를 빠르게 처리합니다. (예: 퀵 정렬, 병합 정렬, 힙 정렬)
- 특수 상황에서의 정렬: 값의 범위가 제한적일 때 비교를 하지 않고 정렬합니다. (예: 계수 정렬, 기수 정렬)

2. 안정성
정렬 전, 값이 같은 요소들이 여러 개 있을 때 그들의 원래 순서가 정렬 후에도 유지되는가를 기준으로 합니다.
- 안정 정렬(Stable) : 중복된 값의 상대적인 순서가 유지됩니다.(예: 삽입 정렬, 병합 정렬, 버블 정렬)
- 불안정 정렬(Unstable) : 중복된 값의 순서가 바뀔 수 있습니다.(예: 퀵 정렬, 힙 정렬, 선택ㄱ 정렬)

3. 제자리 정렬 여부
정렬을 수행할 때 추가적인 메모리 공간을 얼마나 사용하는지를 기준으로 합니다.
- In-place (제자리 정렬) : 주어진 배열 내부에서 위치만 바꾸며 정렬합니다. 메모리 사용량이 매우 적습니다. (예: 퀵 정렬, 힙 정렬, 삽입 정렬)
- Not-in-place (외립 정렬) : 정렬을 위해 원본 배열과 별도의 추가적인 메모리 공간이 필요합니다. (예: 병합 정렬)

4. 비교 방식
데이터를 서로 직접 비교하여 위치를 바꾸는지 여부입니다.

비교 정렬 (Comparison Sort): 두 값을 비교하여 어느 것이 큰지 판단합니다. 

비교 정렬 (Non-comparison Sort): 데이터의 특성(예: 자릿수, 값의 빈도)을 이용하여 비교 없이 정렬합니다. 특정 조건에서 $O(n)$의 속도를 낼 수 있습니다. (예: 기수 정렬, 계수 정렬)
