# JWT란 무엇이며 왜 사용하나요?

## JWT란?
JSON Web Token 당사자 간에 정보를 JSON 개체로 안전하게 전송하기 위한 **개방형 표준**입니다.
주로 인증(Authentication)과 권한 부여(Authorization)를 위해 사용되고, 그 자체가 필요한 정보를 모두 포함하고 있는 **자가 수용적**인 방식이 특징입니다.

## JWT 구조
JWT는 점(.)으로 구분된 세 부분으로 구성됩니다.
1. Header: 토큰의 타입과 사용된 해시 알고리즘 정보를 담고 있습니다.
2. Payload: 전달할 정보(claims)가 담겨 있습니다. 사용자 ID, 유효 기간 등이 포함됩니다.
3. Signature: Header와 Payload를 인코딩한 값과 서버만 알고 있는 **비밀 키(Secret Key)**를 합쳐 생성합니다. 

## JWT를 사용하는 이유
- 무상태성 및 확장성
  - 세션의 한계: 기존 방식은 서버 메모리에 세션 정보를 저장해야 하므로, 사용자가 많아지면 서버 부하가 커지고 여러 대의 서버를 운영할 떄 공유 설정이 복잡합니다.
  - JWT의 해결책은 서버는 토큰을 저장하지 않습니다. 클라이언트가 보낸 토큰을 검증만 하면 되므로 서버 자원을 아낄 수 있고, 서버를 수평적으로 확장하기에 매우 유리합니다.

- 보안성 및 무결성
  - 디지털 서명이 포함되어 있어 클라이언트나 제3자가 토큰 내의 데이터를 변경하면 서버 검증 단계에서 즉시 차단됩니다. 데이터의 신뢰성을 보장합니다.

- 플랫폼 간 호환성
  - 쿠키는 도메인 제한이 엄격하여 여러 도메인 간의 데이터 공유가 어렵지만, JWT는 HTTP 헤더에 담아 보내므로 CORS (Cross-Origin Resource Sharing) 문제에서 자유롭습니다. 특히 쿠키 관리가 복잡한 모바일 앱 환경에서 표준 인증 방식으로 선호됩니다.

## JWT 인증 프로세스
1. 사용자가 로그인 정보(ID/PW)를 서버에 전송
2. 서버는 정보를 확인하고 해당 사용자를 위한 JWT를 생성하고 클라이언트에 응답
3. 클라이언트는 이후 요청 시 **HTTP헤더**에 JWT를 포함시켜 보냄
4. 서버는 클라이언트가 보낸 토큰의 서명을 검증하고, 유효하다면 요청을 처리


JWT는 서버의 상태를 유지하지 않으면서도 보안성을 갖춘 토큰 기반 인증 표준으로, 현대적인 분산 시스템과 모바일 환경에서 높은 확장성을 제공하기 위해 사용.
